# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A static one-page portfolio site for artist Carol Baker-Lai, deployed via GitHub Pages (custom domain in `CNAME`: carolbakerlai.art). No build step, no package manager, no framework — plain HTML/CSS/JS served as-is.

## Running locally

There's no dev server or build command. Open `index.html` directly in a browser, or serve the directory with any static file server (e.g. `python -m http.server`) to avoid `file://` CORS quirks.

## Architecture

- `index.html` — single page with sections: nav, hero, gallery, CTA, about, footer, lightbox modal. Loads `gallery.js` before `script.js`.
- `gallery.js` — **data file, not logic**. Exports `window.GALLERY_DATA`, an array of painting entries (`src`, `title`, `medium`, `dimensions`, `year`, `category`, `notes`). This file is regenerated by `sync_notion.py` — hand edits will be overwritten by the next sync.
- `script.js` — all page behavior: nav scroll/highlight state, building gallery DOM from `GALLERY_DATA`, the lightbox (open/close/prev/next, swipe, keyboard nav), category filter tabs (auto-derived from distinct `category` values in the data), and scroll-reveal animations via `IntersectionObserver`.
- `styles.css` — single stylesheet, custom properties defined in `:root`, sections ordered to mirror `index.html` (nav → hero → gallery/filters/masonry grid → about → lightbox → reveal animations).
- `images/` — painting image files referenced by `gallery.js` `src` paths.

Cache-busting: `index.html` loads `gallery.js?v=2` and `script.js?v=2` — bump the `?v=` query param when shipping changes to either file so GitHub Pages / browsers don't serve stale cached copies.

## Notion sync pipeline

`sync_notion.py` is the source-of-truth updater for gallery content:

- Reads `NOTION_TOKEN` from `.env` (local) or the environment (CI), queries the Notion "Paintings" database (`DB_ID` in the script), filtering to pages with **Include on Website** checked, ordered by the **Order** property.
- For each painting: slugifies the title into a filename, downloads the image into `images/` if not already present (existing files are never re-downloaded/overwritten), and maps Notion properties (Medium, Dimensions, Available for Sale, Category) into gallery entry fields.
- Rewrites `gallery.js` from scratch in title-to-slug order — it is fully regenerated each run, not diffed/merged.
- Run manually: `python sync_notion.py` (needs `NOTION_TOKEN` in `.env`).
- Automated via `.github/workflows/sync.yml`: runs daily at 2am UTC (and on manual `workflow_dispatch`), commits and pushes any resulting changes to `gallery.js`/`images/` directly to `main` as `github-actions[bot]`.

When editing gallery content by hand, prefer updating Notion (the sync will pick it up) over editing `gallery.js` directly, since the next scheduled sync will overwrite manual edits to that file.
